# 测试模块文档

## 概述

本项目使用 Vitest 作为测试框架，提供了完整的测试覆盖，包括单元测试、集成测试和端到端测试。

## 测试结构

```
src/test/
├── setup.ts                    # 测试环境设置
├── unit/                       # 单元测试
│   ├── utils.test.ts          # 工具函数测试
│   ├── auth.test.ts           # 认证模块测试
│   └── database.test.ts       # 数据库模块测试
├── integration/                # 集成测试
│   └── middleware.test.ts     # 中间件集成测试
└── e2e/                       # 端到端测试
    └── login-flow.test.ts     # 登录流程测试
```

## 测试类型

### 1. 单元测试 (Unit Tests)

测试独立的函数和模块，确保每个组件都能正常工作。

#### 工具函数测试 (`utils.test.ts`)
- **密码加密和验证**: 测试 bcrypt 加密和验证功能
- **JWT Token 生成和验证**: 测试 JWT 签名和验证
- **随机字符串生成**: 测试随机字符串生成功能
- **日期格式化**: 测试日期格式化工具
- **客户端IP获取**: 测试IP地址提取功能
- **日志工具**: 测试日志记录功能

#### 认证模块测试 (`auth.test.ts`)
- **登录功能**: 测试用户登录的各种场景
- **Token 生成**: 测试 JWT token 生成
- **组织信息查询**: 测试用户组织关联查询
- **用户权限验证**: 测试角色和权限验证

#### 数据库模块测试 (`database.test.ts`)
- **数据库连接**: 测试数据库连接功能
- **用户表操作**: 测试用户的 CRUD 操作
- **机构表操作**: 测试机构的查询操作
- **角色表操作**: 测试角色的查询操作
- **权限表操作**: 测试权限的查询操作
- **菜单表操作**: 测试菜单的查询操作
- **复杂查询**: 测试多表关联和条件查询

### 2. 集成测试 (Integration Tests)

测试模块间的交互和组合功能。

#### 中间件集成测试 (`middleware.test.ts`)
- **中间件组合**: 测试中间件的执行顺序和错误处理
- **请求验证**: 测试 HTTP 方法验证
- **表单验证**: 测试表单数据验证
- **速率限制**: 测试请求频率限制
- **错误处理**: 测试错误捕获和处理
- **完整流程**: 测试完整的中间件链

### 3. 端到端测试 (E2E Tests)

测试完整的用户操作流程。

#### 登录流程测试 (`login-flow.test.ts`)
- **完整登录流程**: 测试从获取验证码到登录成功的完整流程
- **验证码错误**: 测试验证码验证失败的情况
- **用户不存在**: 测试用户不存在的情况
- **密码错误**: 测试密码错误的情况
- **用户被禁用**: 测试用户被禁用的情况
- **表单验证**: 测试表单字段验证
- **速率限制**: 测试登录频率限制
- **会话管理**: 测试登录后的会话设置
- **错误处理**: 测试网络错误和服务器错误

## 运行测试

### 基本命令

```bash
# 运行所有测试
pnpm test

# 运行测试并生成报告
pnpm test:run

# 运行测试并生成覆盖率报告
pnpm test:coverage

# 监听模式运行测试
pnpm test:watch
```

### 运行特定测试

```bash
# 运行特定测试文件
pnpm test src/test/unit/utils.test.ts

# 运行特定测试套件
pnpm test --grep "工具函数测试"

# 运行特定测试用例
pnpm test --grep "应该正确加密密码"
```

## 测试最佳实践

### 1. 测试命名

- 使用描述性的测试名称
- 遵循 "应该..." 的命名模式
- 清晰表达测试的预期行为

### 2. 测试结构

```typescript
describe('模块名称', () => {
  beforeEach(() => {
    // 设置测试环境
  });

  describe('功能描述', () => {
    it('应该执行特定行为', async () => {
      // 准备测试数据
      const input = 'test';
      
      // 执行被测试的功能
      const result = await functionToTest(input);
      
      // 验证结果
      expect(result).toBe('expected');
    });
  });
});
```

### 3. 模拟 (Mocking)

- 使用 `vi.mock()` 模拟外部依赖
- 使用 `vi.fn()` 创建模拟函数
- 使用 `vi.spyOn()` 监视函数调用

### 4. 测试数据

- 使用 `createMockUser()` 等工具函数创建测试数据
- 避免硬编码测试数据
- 使用有意义的测试数据

### 5. 异步测试

- 使用 `async/await` 处理异步操作
- 使用 `waitFor()` 等待异步结果
- 正确处理 Promise 和错误

## 故障排除

### 常见问题

1. **测试失败**: 检查模拟是否正确设置
2. **覆盖率低**: 添加更多测试用例
3. **异步错误**: 确保正确使用 `async/await`

### 调试技巧

```bash
# 运行单个测试并显示详细输出
pnpm test --reporter=verbose src/test/unit/utils.test.ts

# 在测试中设置断点
pnpm test --inspect-brk
```
